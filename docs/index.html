Frequently Asked Questions

Everything you need to know about Arcile.

Does Arcile collect any of my data?

Absolutely not. Arcile is designed with a strict offline-only architecture. It does not even request the android.permission.INTERNET permission, making it physically impossible for the app to send telemetry, analytics, or personal files to any server.

Is Arcile completely free to use?

Yes, Arcile is 100% free for personal use. The code is publicly visible as source-available software under the custom Tremors Source License (TSL). It contains absolutely no advertisements, trackers, or hidden telemetry.

Does it support external SD cards and USB drives?

Absolutely. Arcile is built to seamlessly read and manage files across your Internal Storage, physical SD Cards, and plugged-in USB drives.

Is Arcile on the Google Play Store?

Not currently. Arcile is exclusively available for download directly from our GitHub Releases page. This allows us to push updates rapidly and bypass restrictive App Store policies that can limit advanced file management capabilities.

What Android versions are supported?

Arcile requires a minimum of Android 7.0 (Nougat / API 24). It is heavily optimized and targeted for the latest Android 14+ architectures.

How does the Trash system work?

Instead of permanent deletion, Arcile routes deleted files into a dedicated hidden folder (`.arcile/.trash` and `.arcile/.metadata`). You can safely review and restore files with their original metadata and paths intact.

Where can I report bugs or request features?

Since Arcile is currently in active Beta development, feedback is highly appreciated! The best place to report bugs or request features is directly on our GitHub repository's issue tracker.
